# План тестирования приложения "Мобильный хоспис"

## Описание приложения:

Приложение даёт функционал по работе с новостями хосписа и включает в себя:

- информацию о новостях и функционал для работы с ними;
- тематические цитаты;
- информацию о приложении.

## Процесс тестирования:

1 составление чек-листа;
2 составление тест-кейсов;
3 ручное тестирование;
4 автоматизация тест-кейсов;
5 отчет о тестировании.

## Виды тестирования:

- Позитивное тестирование;
- Негативное тестирование;
- Функциональное тестирование;
- Системное тестирование
- UX-тестирование
- GUI-тестирование

## Автоматизация:

- авторизация: тест кейс № 1, 2, 3, 4 5, 6, 7, 8, 9.
- панель управления: тест кейс № 24, 25, 26, 27, 28, 29, 30, 31

## Список инструментов для ручного тестирования приложения "мобильный хоспис":

1. Android Studio эмулятор android 10.0, api, экран 2280 х 1080 px; 
2. xiaomi redmi 9t.

## Список инструментов для автоматизации тестирования приложения "мобильный хоспис":

1. Android Studio;

2. Espresso;

3. Allure;

4. Java 11;

5. JUnit 4.13.2.

## Перечень возможных рисков при автомитизации тестирования:

1. изменение функционала приложения (изменение логики добавления, редактирования новостей, добавление ролей пользователя в приложении(гость, администратор));
2. изменение в аторизации (добавление двухфакторной аутентификации, авторизация с помошью Email, vk);

## Предполагаемое время выполнение тестирования:

1. ручное тестирование 106 часов;
2. автомитизация тестирования с учетом рисков 40 часов.
